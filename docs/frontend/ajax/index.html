Asynchronous Javascript and XML

AJAX is a really useful tool that allows you to query a data source and dynamically update it on your page without having to reload. This is imperative for optimal interaction with your user (remember what can happen to our JS when the page reloads?). We can get text, HTML, XML, or JSON data using AJAX. We are going to use jQuery for AJAX because it is far less code than using Vanilla JS, but it can be done. One last thing. Before using AJAX, we must be running a server so I will set up a simple eclipse project so we can demonstrate functionality.

$(element).load()

The .load(data) method allows you to grab data from a specific location and load it back to an element in your HTML. This is a very basic AJAX call as it only accepts a parameter that is the location of the data you want to access. We can see this in action here:

$('#myInfo').load('some-html-element.html');

$.get()

shorthand method. The $.get() method is a little more robust as it accepts some more specific parameters as well as a callback function to execute some functionality once a response comes back. Let’s see how to use this method.

// The location you want to query
const url = '/courses';
// This will get appended to the url as
// a query string '?courseName=javascript'
const data = {
    courseName: javascript;
};
// What you want to do when the response comes back
const callback = function(response) {
    $($('main')).append('<p>' + response + '</p>');
}
$.get(url, data, callback);

There are many more shorthand methods. I encourage you to check out the jQuery docs to see what else you can do.

$.ajax()

The $.ajax(settings) method uses http requests to grab data from a server and sends it back to your document. All of this without reloading your page! Notice how we call this directly on the jQuery object as well.

$.ajax({
    type: GET,
    dataType: "json",
    url: "./data/data.json",
    success: function(res) {
        $('h1').text(res.title);
    }
});

Again, check the docs for a full list of what this method can do.

Conclusion

There are many other shorthand methods for ajax as well as many other key/value pairs you can pass to the $.ajax() method. AJAx become a lot more powerful when you can pull data from other APIs such as Twitter, Facebook, Google Maps, Flickr, and countless others. Remeber that AJAX always needs a server running to function. if there is no server, you can’t make HTTP requests. If you feel like getting some practice, use the methods listed here with an API that requires no authentication to use. I like this one.
