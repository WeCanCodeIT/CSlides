Equality operators

  • We have seen how we use the = operator to assign a value to a variable.

  • I think of the = operator in the terms of ownership. Lets review the following line of code.

    • ageTwo = 70;

    • By using the = operator, we can say that ageTwo owns the value 70.

  • We also have another catagory of operators which use the = operator. We use the following equality operators when we need to make decisions on the flow of logic.

  • It is important to note using and equality operator will not return a numeric value. The equality operator will instead return a value of True or False.

  • We will see plenty of examples in upcoming classes which will demonstrate how to use equality operators. For our purposes I will highlight a few examples.

    • The == operator tests for equality. An example would be (5 == 5). We know 5 is equal to 5, so the value returned will be True.

    • Another example would be (5 == 56). We know 5 is not equal to 56, so the value returned will be False.

    • Yet another example would be (‘5’ == 5). Can someone guess what the return value will be. Why?

Inequality

  • We can also check for inequality with the inequality operator !=.

  • We will see plenty of examples in upcoming classes which will demonstrate how to use equality operators. For our purposes I will highlight a few examples.

    • In programming, we commonly use the ! as a not operator.

    • The != operator tests for inequality. An example would be (5 != 5). We know 5 is equal to 5, so the value returned will be False.

    • Another example would be (5 != 56). We know 5 is not equal to 56, so the value returned will be True.

    • Yet another example would be (‘5’ != “five”). Can someone guess what the return value will be. Why?

Relational operators

  • If we think back again to our early education years, we will remember using the following operators.

    • We would use the < > operator to check if value a is less than or greater than value b.

    • So a < b checks to see if a is less than value b.

    • Likewise, a > b checks to see if a is greater than value b.

  • Now lets incorporate our = operator.

    • So a <= b checks to see if a is less than or equal to value b.

    • Likewise, a >= b checks to see if a is greater than or equal to value b.

  • Finally lets incorporate our ! operator.

    • So a !< b checks to see if a is not less than value b.

    • Likewise, a !> b checks to see if a is not greater than value b.

Translating English to CSharp

Let’s translate English statements into C#. First, we’ll do one together.

Start by writing the statements as comments. Here’s an example:

// Jessica is 23 years old.
int jessicaAge = 23;

// Sam is 47.
int samAge = 47;

// Jessica is younger than Sam.
Console.WriteLine(jessicaAge < samAge);

Let’s break it down…

Defining a variable to hold Jessica’s age:

We need to create a variable to hold the age of Jessica. What should its type be?

// Jessica is 23 years old.

Since we think of ages as whole numbers, we will use an int:

int jessicaAge

Let’s break it down…

Assigning a value for Jessica’s age:

The word “is” means equals. How do we represent assignment?

// Jessica is 23 years old.

If Jessica "is 23", then Jessica's age = 23:

int jessicaAge = 23;

Let’s break it down…

Comparing Jessica’s and Sam’s ages:

To say Jessica is younger than Sam means we are comparing their ages. Which operator would we use to compare them?

// Jessica is younger than Sam.

We use the "less than" relational operator to perform this comparison:

Console.WriteLine(jessicaAge < samAge);

Let’s look at the syntax

Single line comments start with //:

// Jessica is 23 years old.
int jessicaAge = 23;
// Sam is 47.
int samAge = 47;
// Jessica is younger than Sam.
Console.WriteLine(jessicaAge < samAge);

Let’s look at the syntax

Each statement ends with a semicolon. It acts like a period (.) in English at the end of a sentence.

// Jessica is 23 years old.
int jessicaAge = 23;
// Sam is 47.
int samAge = 47;
// Jessica is younger than Sam.
Console.WriteLine(jessicaAge < samAge);

The statement:

int jessicaAge = 23;

reads as “Jessica’s age is 23.”

Let’s look at the syntax

Console.WriteLine is a method, so it ends with a set of opening and closing parentheses:

Console.WriteLine(jessicaAge < samAge);

Inside the parentheses, we find the method arguments. For the println method, this is what we want to print. In this case, that's whether Jessica is younger than Sam:

Console.WriteLine(jessicaAge < samAge);
